# 纯接触盲走迷宫 — 方案设计文档

> **⚠️ 已被取代**：2026-07-02 完成了 9 路深度调研后，技术方案和路线图已全面升级，
> 见 **`docs/blind_maze_research_and_method.md`**（调研综述 + 方案对比 + method + 代码路线）。
> 本文保留作为初版思路记录。

> **终极目标**：让 G1 人形机器人在**没有视觉、没有雷达、没有地图**的情况下，
> **只靠自身不断的碰撞感知**（本体感觉估计的全身接触），在迷宫里探索并走出来。
> **最终部署在真机上**（真机没有接触/力传感器，本体感觉估计就是刚需）。
>
> 分支 `wholebodycontact`。本文档是路线图，会随进展更新。

---

## 0. 一句话可行性结论

**概念成立、有明确研究价值，但这是一个多阶段研究项目，且因"真机 + 纯接触"两个约束
而处于/接近研究前沿。** 现有 Plan A（本体感觉力估计）只是最底层的一块基石，而且
**还不能直接用**——它是在"静态站立 + 离散点力"上训的，和"行走撞墙"的分布不同。

可行的前提是**严格按生死判定关卡分阶段推进**，每个阶段用最便宜的实验先证明"这条路
通不通"，再投入。

## 1. 目标与约束

| 项 | 内容 |
|---|---|
| 任务 | 在未知迷宫中探索并抵达出口 |
| 感知 | **仅**全身接触（撞到哪、大致法向）——纯盲走 |
| 感知来源 | **本体感觉**（关节位置/速度/力矩、IMU），**不用**任何接触/力传感器 |
| 平台 | 最终**真机** G1（先仿真验证，再 sim-to-real） |
| 已有 | 冻结的低层 locomotion 策略；Plan A 本体感觉力估计（区域级） |

**"真机 + 纯接触"意味着什么：**
- 真机上没有 ground-truth 接触可用 → 感知**必须**靠本体感觉，且必须扛过 sim-to-real。
- 纯接触 → **没有退路**，感知一旦失灵，导航就瞎；不像有视觉时接触只是补充。
- → 感知的**鲁棒性**是整个项目的命门。

## 2. 为什么难 —— 风险清单

| 风险 | 说明 | 与现状的关系 |
|---|---|---|
| **① 行走中感知接触** | 行走时本体信号噪声远大于站立；撞墙信号更弱、更瞬态 | Plan A 是静态下测的，动态只会更差 |
| **② 撞墙部位是弱区** | 走路撞墙主要撞躯干/髋/膝——正是定位最差的区域（recall 0.25–0.32） | 手臂虽准（0.7），但要主动"伸手当探须"才用得上 |
| **③ 接触分布不匹配** | net_pull 是离散点的拉力(10–40N)；撞墙是刚性面的持续反作用力 | **必须重新采集"行走撞墙"数据**，现模型不可直接迁移 |
| **④ 导航探索策略** | 用带噪接触流去探索+走出迷宫，是独立的 RL 难题 | 目前完全没有 |
| **⑤ sim-to-real** | 接触动力学、本体信号分布在真机上都会漂移；纯接触无冗余 | 尚未涉及 |

**两个降低难度的关键洞察（贯穿全案）：**
1. **导航不需要精确定位**，只需要粗糙的**"前/左/右/后哪边撞到 + 大致法向"**就够做
   **沿墙走（wall-following）**。把感知目标从"5 解剖区域 / 3D 力向量"**降级为 4~6 向接触**，
   可行性大幅提升。
2. **让机器人张开手臂当"触须"摸墙**——正好用上"手臂定位最强"这个长处，像盲人探路。

## 3. 系统架构（三层）

```
┌─────────────────────────────────────────────┐
│  高层：导航 / 探索策略  (待建, RL)            │
│  输入: 接触方向估计 + 本体 + 记忆            │
│  输出: 行走方向 / 转向命令                    │
├─────────────────────────────────────────────┤
│  中层：接触感知  (Plan A 的演化)             │
│  本体感觉 → "撞到哪一侧 + 大致法向"          │
│  ★ 需针对"行走撞墙"重训, 标签重新定义        │
├─────────────────────────────────────────────┤
│  低层：locomotion  (已有, 冻结)              │
│  行走命令 → 关节动作, 维持平衡                │
└─────────────────────────────────────────────┘
```

**解耦原则**：感知（中层）和导航（高层）**分开验证**。先各自用"理想输入"证明能work，
再串起来。这样任何一层失败都能快速定位。

## 4. 感知的重新定义（从 Plan A 到撞墙）

Plan A 现状 → 迷宫任务需要的改动：

| 维度 | Plan A 现在 | 迷宫任务需要 |
|---|---|---|
| 力来源 | net_pull 离散点拉力 | 行走中撞刚性墙的反作用力 |
| 机器人状态 | 站立/跟踪(近静态) | 动态行走 |
| 标签 | 5 解剖区域 + 3D 力向量 | **接触方向**（前/左/右/后）+ 粗法向 |
| 目的 | "身上哪被推" | "该往哪躲 / 墙在哪边" |

→ **Phase 0 的核心就是完成这个重定义 + 重新采集/训练。**

## 5. 分阶段路线图（含生死判定关卡）

> 原则：**便宜的 make-or-break 检查先做。** 每个 Phase 结尾有 **Go/No-Go 关卡**。

### Phase 0 — 感知能否用于"行走撞墙"？ 【生死判定 1】
- 搭简单场景：机器人**边走边撞**墙/障碍；从仿真读 ground-truth 接触（哪一侧+法向）当标签。
- 输入沿用 Plan A 的本体感觉组（`wbc_input_`），**标签换成 4~6 向接触方向 + 粗法向**。
- 训练 + 评估：**动态行走下能否可靠检测 + 粗定位撞墙方向？**
- **Go 条件**：接触检测召回 > ~0.8、方向 4 分类准确率 > ~0.7（阈值待定）。
- **No-Go 应对**：让机器人伸手当触须重采；或降级到"只检测有无接触，不定方向"。

### Phase 1 — 用"完美接触"跑通导航？ 【生死判定 2】
- Isaac Lab 里搭**参数化迷宫**环境（可随机生成）。
- 训高层导航策略（RL）：输入 = **仿真真值接触** + 本体 + 简单记忆(如已走轨迹)，
  输出 = 行走方向/转向；奖励 = 靠近出口 + 探索覆盖 - 卡住惩罚。
- **先证明：给"上帝视角的完美接触"，机器人能不能走出迷宫？**
- **Go 条件**：随机迷宫里成功率 > ~某阈值（如 70%）。
- **No-Go 应对**：完美接触都走不出 → 问题在导航算法（探索/记忆/奖励），感知再好没用，
  先专攻导航（可考虑经典 wall-following 作为 baseline 对照）。

### Phase 2 — 换成本体感觉估计（感知↔导航串联）
- 把 Phase 1 的真值接触替换成 Phase 0 的 MLP 估计。
- 测性能掉多少；必要时**用带噪接触重新微调/联合训练**导航策略（让它学会容忍误报/漏报）。
- **Go 条件**：用估计接触的成功率相对完美接触下降可接受（如掉幅 < 某阈值）。

### Phase 3 — 仿真鲁棒性 + 真机准备
- 域随机化（摩擦、质量、延迟、传感噪声、迷宫布局）。
- 真机数据核对：在真机上采一段"撞墙"本体数据，检查 Phase 0 模型的估计是否成立
  （**sim-to-real 的第一道坎，越早查越好**）。

### Phase 4 — 真机部署
- sim-to-real 落地；纯接触无冗余，需保守的安全策略（撞太狠就停、跌倒保护）。

---

## 6. 真机专项考量（因为终点是真机，越早想越好）

- **本体感觉估计必须扛过 reality gap**：真机的力矩估计、关节摩擦、传感延迟都和仿真不同。
  建议 Phase 0 就用域随机化训，并尽早（Phase 3 前）在真机采一小段撞墙数据做核对。
- **纯接触 = 无冗余**：任何一次漏检都可能让机器人卡死或跌倒。导航策略要对"感知不可靠"
  鲁棒（Phase 2 的带噪训练是关键）。
- **安全**：真机反复撞墙有硬件风险。需要力上限、跌倒保护、撞击缓冲策略。
- **"伸手触须"策略**在真机上更有价值（手臂信号最干净），但要保证伸手不影响行走平衡。

## 8. 待定决策（需要拍板的地方）

- **接触方向的粒度**：4 向（前后左右）够不够？还是要 6~8 向 + 粗法向？
- **要不要"伸手触须"**：主动伸手摸墙 vs 被动撞到才知道？前者好感知、但要改行走姿态。
- **导航的记忆机制**：纯反应式 wall-following？还是给它某种内部地图/占据栅格记忆？
- **迷宫复杂度**：先从最简单的直角走廊，还是一开始就随机迷宫？
- **是否允许"轻微视觉"作对照实验**：纯盲走是目标，但训练期可否用真值加速？（当前定：纯接触）
